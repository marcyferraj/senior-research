# Performance Comparison: Monolithic vs. Microservice Architectures

## Project Objective
The objective of this research is to evaluate the performance trade-offs between monolithic and microservice architectures. Specifically, measuring and comparing metrics such as fault rates and concurrency limits in an e-commerce backend.

## Background
As modern applications scale, many developers transition from a "Monolith" (single, unified unit) to "Microservices" (independent, specialized services). This project uses a simulated e-commerce store built in Python and PostgreSQL to determine the specific points where architectural complexity provides a measurable advantage.

## Tools & Technologies
* **Framework:** Python
* **Database:** PostgreSQL 
* **Environment:** Docker 
* **Testing:** Performance benchmarking for concurrency and fault tolerance
